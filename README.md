MERO — Магазин сучасного одягу
MERO — це повнофункціональний прототип інтернет-магазину, розроблений у межах курсу вебпрограмування (Практичні роботи №1-5). Проєкт демонструє поєднання семантичної верстки, адаптивного дизайну та динамічного рендерингу даних.

 Основні можливості
Динамічний каталог: Автоматична генерація карток товарів із файлу data.json за допомогою Fetch API.

Адаптивність (Mobile-First): Повна підтримка мобільних пристроїв, планшетів та десктопів завдяки CSS Grid (патерн 4-3-2-1).

Інтерактивність: Робоче бургер-меню, система «лайків» та вибір параметрів товару.

Валідація форм: Просунута клієнтська перевірка номера телефону за допомогою регулярних виразів та візуальний фідбек через CSS псевдокласи.

Управління станом: Збереження обраних товарів у кошику через localStorage.

🛠 Технологічний стек
HTML5: Семантична розмітка (header, nav, main, section, article).

CSS3: Глобальні змінні (:root), Grid Layout, Flexbox, анімації transition.

JavaScript (Vanilla): Асинхронні функції (async/await), Fetch API, маніпуляції з DOM, Template Literals.

 Структура проєкту
index.html — головна сторінка з Hero-секцією.

catalog.html — динамічний каталог товарів.

product.html — сторінка окремого товару з вибором розміру.

cart.html — сторінка оформлення замовлення та кошик.

style.css — стилі проєкту (Mobile-First архітектура).

data.json — база даних товарів.

 Вимоги до розробки (Compliance)
Проєкт відповідає наступним критеріям якості:

Валідність: Відсутність помилок у структурі тегів, наявність alt для всіх зображень.

Якість стилів: Використання CSS-змінних, стратегія Mobile-First, відсутність дублювання коду.

Надійність скриптів: Ефективне використання асинхронних функцій та обробка станів завантаження.

UX/UI: Візуальний відгук на дії користувача (hover, active), стабільність верстки без горизонтального скролу.

 Як запустити
Оскільки проєкт використовує Fetch API для завантаження даних, його необхідно запускати через локальний сервер.
Рекомендовано: Використовувати розширення Live Server у Visual Studio Code. При простому відкритті HTML-файлу через провідник браузер може заблокувати запити через політику CORS.
